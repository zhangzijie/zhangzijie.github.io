SQL语句中字符串前加N解决中文乱码

之前给Oracle数据库插入数据时，插入中文字符串后都变成乱码了。解决方法是在字符串前加大写的N。

加N表示字符串强制用 Unicode 方式存储。

Unicode 字符串的格式与普通字符串相似，但它前面有一个 N 标识符（N 代表 SQL-92 标准中的国际语言 (National Language)）。N 前缀必须是大写字母。

Unicode是双字节字符。对于西文字符，用一个字节来存储过足够了，对于东方文字字符，就需要两个字节来存储。Unicode为了统一、规范、方便、兼容，就规定西文字符也用两个字节来存储。

微软也有相关的帮助文档：当您在 SQL Server 中的 Unicode 字符串常量处理，则必须在前面带有前缀 N 的所有 Unicode 字符串

当然这里的解决方法除了在字符串前加N，也可以通过更改数据库的字符集来解决。
